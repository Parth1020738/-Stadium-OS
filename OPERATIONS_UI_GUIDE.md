# Operations Component & UI/UX Style Guide

This document describes the design conventions, components layout, and color palettes.

---

## 1. Palette & Status Codes

We enforce consistent color coding across tables, lists, and drawers:
*   **CRITICAL / HIGH / DELAYED**: `text-red-500 bg-red-500/10 border-red-500/20`
*   **WARNING / ASSIGNED / IN TRANSIT**: `text-yellow-500 bg-yellow-500/10 border-yellow-500/20`
*   **NOMINAL / COMPLETED / STABLE**: `text-emerald-500 bg-emerald-500/10 border-emerald-500/20`

---

## 2. Shared Interface Layouts

*   **KPI stats row**: Standard 3-column statistics grid utilizing lucide icons.
*   **Actions toolbar**: Unified filter select dropdowns and search bars.
*   **Workspace Splitting**: Two-column layout grid. Left column displays lists or tables; right column renders detail cards, timelines, or configuration controls.
